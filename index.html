Mode Simpel: Restart cepat (paling sering dipakai)

Target: server habis reboot, atau aplikasi terasa “ngadat”. Fokus: nyalain semua, cek cepat, selesai.

1
Start / Rebuild stack (normal)

Kalau ada perubahan code/image, pakai --build.

Terminal 
cd ~/docker-stack
sudo docker compose up -d --build
2
Cek container up dan port terbuka
Terminal 
sudo docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

Wajib terlihat: rfid-frontend 5173, rfid-proxy 8000, django-gm1 7000.

3
Smoke test endpoint (3 detik)
Terminal 
curl -m 3 -I http://10.5.3.195:5173 | head
curl -m 3 -I http://10.5.3.195:8000 | head
curl -m 3 -sS http://10.5.3.195:8000/wira | head
Normal: akses http://10.5.3.195:7000/wira langsung bisa 403 karena butuh API key, tapi via proxy :8000 harus 200.
4
Restart satu service saja (kalau error spesifik)
Restart proxy
cd ~/docker-stack
sudo docker compose up -d --no-deps --force-recreate proxy
Restart backend
sudo docker restart django-gm1
Restart mysql
sudo docker restart mysql-server
Restart mosquitto
sudo docker restart mosquitto
5
Kalau masih bermasalah, ambil log 60 baris
Terminal
sudo docker logs --tail=60 rfid-proxy
sudo docker logs --tail=60 django-gm1
sudo docker logs --tail=60 mysql-server
sudo docker logs --tail=60 mosquitto
sudo docker logs --tail=60 mqtt-listener

Mode Simpel: Stop aman

Menghentikan stack compose tanpa menghapus volume eksternal seperti mysql_data.

Terminal
cd ~/docker-stack
sudo docker compose down

Masalah yang paling sering (dan artinya)

Biar kamu cepat “nembak” sumber masalah tanpa muter-muter.
Backend /wira 403 Forbidden
Itu normal kalau akses langsung ke :7000 tanpa header X-Api-Key. Solusi: akses via proxy :8000 atau kirim header API key dari Postman.
Proxy timeout ke backend (ConnectTimeout / UND_ERR_CONNECT_TIMEOUT)
Biasanya proxy masih mengarah ke IP lama (misal 10.5.3.195:7000) padahal harusnya bisa pakai service name. Pastikan environment: BACKEND_URL=http://django-gm1:7000. Restart proxy.
MySQL denied (ER_ACCESS_DENIED_ERROR)
Password/user tidak sama. Di server.js lama ada password satu1 untuk root, tapi stack MySQL pakai 123456. Fix: pakai env MYSQL_USER/MYSQL_PASSWORD dari compose, dan pastikan code server.js tidak override.
Mosquitto tidak bisa load config file
Biasanya mount path salah atau permission folder tidak bisa diakses. Gunakan path absolut dan pastikan file ada: /home/robotics/mosquitto/config/mosquitto.conf.
mqtt-listener "Connection refused" ke mosquitto
Broker belum up, salah host/port env, atau container belum join network. Cek log mosquitto, lalu test TCP dari mqtt-listener container.
Ringkasnya: Akses UI lewat :5173, API lewat Proxy :8000, Backend :7000 jangan dites tanpa API key (403 normal).
