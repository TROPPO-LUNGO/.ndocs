Quick Start (paling sering dipakai)

Ini langkah singkat ketika server habis reboot atau kamu mau jalankan semuanya dari nol menggunakan docker compose.

1) Jalankan stack

cd ~/docker-stack
sudo docker compose up -d --build
--build dipakai kalau ada perubahan kode/image.
2) Cek status container

sudo docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
Pastikan semua service penting Up dan port terbuka.
3) Smoke test endpoint

curl -m 3 -I http://10.5.1.148:5173 | head
curl -m 3 -I http://10.5.1.148:8000 | head
curl -m 3 -i http://10.5.1.148:8000/wira | head
Proxy harus bisa akses backend dengan header API key (proxy yang nambahin).
4) Verifikasi backend auth normal

# Ini akan 403 (NORMAL) karena tanpa API key header
curl -m 3 -i http://10.5.1.148:7000/wira | head -n 12

# Ini harus 200 karena proxy menambahkan header X-Api-Key
curl -m 3 -i http://10.5.1.148:8000/wira | head
403 di backend langsung bukan error, itu tanda proteksi jalan.

1. Overview Arsitektur

Stack ini terdiri dari 3 lapisan utama:

A) Data & Messaging Layer
  • MySQL: database utama (user, garment, tracking_movement, dll).
  • Redis: channel layer untuk Django Channels (WebSocket).
  • Mosquitto: MQTT broker untuk event RFID (topic rfid/gt/id).
B) Backend Layer
  • Django ASGI (daphne): API + WebSocket endpoint.
  • move_worker: worker loop memindahkan data DONE ke tabel tracking_movement_end.
  • mqtt_listener: subscriber MQTT, menerima event lalu update DB/backend logic.
C) Frontend Layer
  • Frontend (Vite): UI dashboard di port 5173.
  • Proxy Node (Express): port 8000, jadi “gerbang” ke backend karena dia otomatis menambahkan API key header.
Alur request (yang benar)
  1. User buka UI: http://10.5.1.148:5173
  2. UI akses API ke Proxy: http://10.5.1.148:8000
  3. Proxy meneruskan ke Backend: http://django-gm1:7000 sambil menambahkan X-Api-Key
  4. Backend akses DB MySQL dan Redis, dan mqtt_listener terima event dari Mosquitto

2. Istilah & Pengertian Singkat

docker compose

Cara menjalankan banyak container sekaligus dari satu file docker-compose.yml. Tujuannya supaya tidak mengetik perintah docker run panjang-panjang.

container

Instance yang sedang berjalan dari sebuah image. Contoh: django-gm1 adalah container backend.

image

“Template” untuk membuat container. Contoh: garment-backend:1.0.

volume

Storage persistent. DB MySQL biasanya wajib pakai volume supaya data tidak hilang saat container dihapus.

network (bridge)

Jaringan internal antar container. Dengan network yang sama, container bisa saling panggil pakai nama service, misal mysql-server atau django-gm1.

ports mapping

Menghubungkan port container ke port host. Contoh: 7000:7000.

Daphne / ASGI / Channels

Daphne adalah server ASGI. Django Channels butuh Redis sebagai channel layer untuk WebSocket. Tanpa Redis, WS error.

Proxy API

Server Node (Express) yang meneruskan request ke backend sambil menambahkan API key, serta punya beberapa fitur tambahan.

4. Cara Start / Stop / Restart

Start semua service

cd ~/docker-stack
sudo docker compose up -d --build

Gunakan ini setelah reboot server. Kalau tidak ada perubahan kode, boleh tanpa --build.

Stop semua service stack (tanpa hapus data volume)

cd ~/docker-stack
sudo docker compose down

Ini akan menghapus container compose, tapi volume eksternal (misal mysql_data) tetap aman.

Restart 1 service saja

cd ~/docker-stack
sudo docker compose up -d --no-deps --force-recreate proxy
# atau
sudo docker restart rfid-proxy

Kalau kamu cuma ubah server.js proxy, restart proxy saja.

Lihat log

sudo docker logs --tail=200 rfid-proxy
sudo docker logs --tail=200 django-gm1
sudo docker logs --tail=200 mysql-server
sudo docker logs --tail=200 mosquitto

5. Tahapan Pengecekan Menyeluruh (Checklist)

A) Status container dan port

sudo docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
  • Pastikan django-gm1, rfid-proxy, rfid-frontend status Up.
  • Pastikan port 5173, 8000, 7000 terlihat mapping ke 0.0.0.0.
B) Health test dari host

curl -m 3 -I http://10.5.1.148:5173 | head
curl -m 3 -I http://10.5.1.148:8000 | head
curl -m 3 -I http://10.5.1.148:7000 | head

Backend boleh 200 untuk root, tapi endpoint tertentu butuh API key sehingga bisa 403.

C) Pastikan proxy bisa akses endpoint protected

# Akses backend langsung tanpa header: 403 (normal)
curl -m 3 -i http://10.5.1.148:7000/wira | head -n 12

# Akses via proxy: 200 (wajib)
curl -m 3 -i http://10.5.1.148:8000/wira | head
D) Cek dari dalam proxy container (DNS + backend)

sudo docker exec -it rfid-proxy sh -lc "getent hosts django-gm1 mysql-server redis mosquitto"
sudo docker exec -it rfid-proxy sh -lc "env | grep -E 'BACKEND|MYSQL_'"

Kalau DNS tidak resolve, berarti beda network atau service tidak join network.

E) Cek Django DB dan Redis

sudo docker exec -it django-gm1 sh -lc "python manage.py shell -c \"from django.db import connection; connection.cursor().execute('SELECT 1'); print('DJANGO DB OK')\""
sudo docker exec -it django-gm1 sh -lc "python -c \"import socket; s=socket.socket(); s.settimeout(2); s.connect(('redis',6379)); print('REDIS TCP OK')\""
F) Cek Mosquitto config & port

sudo docker logs --tail=80 mosquitto
sudo docker exec -it mosquitto sh -lc "cat /mosquitto/config/mosquitto.conf | head -n 20"

Kalau muncul Unable to open config file, biasanya path mount atau permission folder.

G) Cek mqtt-listener bisa connect broker

sudo docker logs --tail=120 mqtt-listener
sudo docker exec -it mqtt-listener sh -lc "python -c \"import socket; s=socket.socket(); s.settimeout(2); s.connect(('mosquitto',1883)); print('MQTT TCP OK')\""

Jika masih refused, cek apakah env MQTT_BROKER sudah benar dan broker benar-benar up.

6. Troubleshooting (Kasus umum & solusi)

Kasus 1: Backend /wira 403

Ini normal kalau kamu akses backend langsung tanpa API key header. Solusinya bukan “benerin auth”, tapi pastikan UI pakai proxy atau Postman kirim header X-Api-Key.

Kasus 2: Proxy timeout ke backend (UND_ERR_CONNECT_TIMEOUT)
  • Biasanya karena proxy masih mengarah ke IP lama (10.5.1.148) padahal seharusnya ke service name django-gm1.
  • Fix: set BACKEND_URL=http://django-gm1:7000 di compose.
  • Restart proxy: sudo docker compose up -d --no-deps --force-recreate proxy.
Kasus 3: MySQL Access Denied (ER_ACCESS_DENIED_ERROR)

Ini murni mismatch user/password. Di server.js kamu dulu ada password root satu1, sementara MySQL root password yang dipakai stack ini 123456. Fix: samakan password di environment proxy atau ubah server.js agar membaca dari env.

Kasus 4: Mosquitto "Unable to open config file"
  • Pastikan file ada di host: /home/robotics/mosquitto/config/mosquitto.conf
  • Pastikan mount benar dan permission folder bisa diakses.
  • Paling aman: gunakan path absolut di compose (yang kamu sudah lakukan).
Kasus 5: Redis ConnectionRefused

Biasanya karena backend memakai host 10.0.0.1 atau IP random. Fix yang benar adalah gunakan service name: ("redis", 6379) di CHANNEL_LAYERS.

7. Cleanup (Images, Containers, Volumes) Aman

Kamu punya image yang tidak dipakai lagi seperti garment-tracking:1.0 (ID berbeda), dan beberapa image duplikat rfid_tracking_v2_*. Ini aman dibersihkan untuk hemat disk, selama tidak dipakai oleh container yang sedang up.
A) Hapus image yang jelas tidak dipakai

# Contoh: garment-tracking:1.0 tidak dipakai oleh container manapun
sudo docker image rm garment-tracking:1.0

# Hapus image duplikat (cek dulu: docker ps menggunakan yang mana)
sudo docker image rm rfid-proxy:1.0 rfid_tracking_v2-proxy:latest rfid_tracking_v2_proxy:latest || true
sudo docker image rm rfid_tracking_v2-frontend:latest rfid_tracking_v2_frontend:latest rfid_tracking_v2_rfid-frontend:latest || true

Kalau ada error “image is being used”, berarti masih dipakai container. Stop/hapus containernya dulu.

B) Bersihkan dangling images

sudo docker image prune
C) Bersihkan volume yang benar-benar tidak dipakai

Jangan hapus mysql_data kalau itu menyimpan data produksi. Yang aman dibersihkan biasanya volume hash acak yang tidak terpakai. 


# Lihat volume yang dipakai container
sudo docker volume ls
sudo docker volume prune

docker volume prune hanya hapus volume yang tidak terpakai oleh container manapun.

8. Catatan Keamanan (yang perlu kamu tahu)

  • Port 3306 (MySQL) terbuka ke 0.0.0.0. Kalau ini server production, ini risk. Idealnya batasi ke LAN tertentu atau tidak dipublish (cukup internal docker network).
  • API key jangan ditaruh hardcode di repo publik. Minimal taruh di env / secret. Saat ini proxy menambah header API key, itu benar, tapi tetap rahasiakan key-nya.
  • Worker move_data loop tanpa henti. Pastikan ada log/monitor agar kalau error tidak silent.
